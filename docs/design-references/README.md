# BlockRoom Design References

The previous BlockRoom visual rules have been retired. No reference in this
directory is selected as the final visual direction yet.

Downloaded from
[VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md):

- `cohere/DESIGN.md` — Cohere design-system analysis
- `elevenlabs/DESIGN.md` — ElevenLabs design-system analysis

For the next visual refactor:

1. Read `docs/project-blueprint.md` first. It remains the source of truth for
   product behavior, real data, wallet identity, realtime rooms, media,
   sessions, and dashboard records.
2. Read both reference files in full.
3. Do not assume either reference wins, or mix them, until the product owner
   gives a visual direction.
4. Preserve all current functionality while replacing the visual system.

The existing application remains visually unchanged until that instruction is
given.
